Dein eigenes ChatGPT — lokal und ohne Cloud

Setup-Guide · 5 Tools · Out Of The Box Science

Jede Frage an ChatGPT, Gemini oder Claude wandert auf fremde Server. Dort wird sie gespeichert, oft zum Training genutzt und zu einem Profil über euch verdichtet. Für unveröffentlichte Daten, Interview-Transkripte oder alles unter Schweigepflicht ist das ein echtes Problem. PewDiePie hat mit Odysseus gezeigt, wie einfach es inzwischen ist, eine komplette ChatGPT-artige Oberfläche auf der eigenen Hardware laufen zu lassen — Chat, Agenten, Web-Suche, Gedächtnis. Dieser Guide zeigt euch, welche Werkzeuge ihr braucht, wie das Setup in vier Schritten läuft und wo die ehrlichen Grenzen liegen.

Das Prinzip

Was lokale KI bedeutet

Eine gehostete KI wie ChatGPT ist im Kern zwei Dinge: ein Sprachmodell, das die Antworten erzeugt, und eine Oberfläche drumherum, die Chats, Suche und Gedächtnis verwaltet. Beides läuft auf den Servern des Anbieters. Lokale KI dreht genau das um — das Modell läuft auf eurem Rechner, die Oberfläche ebenfalls, und keine eurer Eingaben verlässt das Gerät.

Warum das für die Wissenschaft zählt: Wer eine vertrauliche Studie, Patientendaten oder ein laufendes Manuskript in ein Cloud-Tool kippt, gibt die Kontrolle darüber ab. Lokal bleibt die Datenhoheit bei euch — DSGVO-konform per Architektur, nicht per Versprechen in den AGB.

Werkzeuge

Was ihr braucht

Odysseus Open Source

Was es ist: Die Oberfläche — ein selbst-gehostetes KI-Workspace mit Chat, Agenten-Modus, Web-Suche, Gedächtnis, Notizen und Deep Research. PewDiePies Projekt, MIT-Lizenz.

Wofür: Euer „ChatGPT-Fenster", nur eben auf eurem eigenen Server.

Wie: github.com/pewdiepie-archdaemon/odysseus

Docker Kostenlos

Was es ist: Der „Motor", der Odysseus und seine Hilfsdienste in Containern startet — ein Befehl statt manueller Installation.

Wofür: Läuft auf Windows, macOS und Linux gleich. Der einfachste Weg, alles zum Laufen zu bringen.

Wie: Docker Desktop installieren.

LM Studio Kostenlos

Was es ist: Ein Programm mit Oberfläche, das lokale Modelle herunterlädt und als Server bereitstellt. Der einsteigerfreundliche Weg.

Wofür: Modell aussuchen, laden, Server starten — Odysseus klinkt sich dann ein.

Neu: Mit LM Link erreicht ihr eure Modelle jetzt auch aus der Ferne, Ende-zu-Ende-verschlüsselt (mehr dazu unten).

Wie: lmstudio.ai · Alternative ohne Oberfläche: Ollama (schlank, per Terminal).

SearXNG Open Source

Was es ist: Eine eigene Meta-Suchmaschine, die Odysseus automatisch mitstartet. Gibt eurem Modell Zugriff aufs Web, ohne über Google zu laufen.

Wofür: Aktuelle Infos und Quellen — die Suche bleibt anonym und unter eurer Kontrolle.

Wie: Kommt im Docker-Setup von Odysseus mit, kein eigener Schritt nötig.

Zur Hardware: Die Oberfläche selbst ist leichtgewichtig — anstrengend wird nur das Modell. Kleine Modelle (7–8 Mrd. Parameter) laufen auf einer GPU mit rund 8 GB VRAM oder einem Apple-Silicon-Mac. Mehr Kontext und größere Modelle brauchen mehr Speicher: Im Video flossen rund 22 GB VRAM für ein größeres Modell mit langem Kontextfenster. Faustregel: Je mehr Kontext ihr wollt, desto mehr Speicher kostet es.

Setup

In vier Schritten zum eigenen KI-Workspace

1.Docker installieren

Ladet euch Docker Desktop herunter und startet es. Damit habt ihr den Container-Motor — den Rest erledigen ein paar Zeilen im Terminal.

2.Odysseus holen und starten

Im Terminal das Projekt klonen und hochfahren. Der letzte Befehl baut alles und lässt es im Hintergrund laufen:

git clone https://github.com/pewdiepie-archdaemon/odysseus.git
cd odysseus
cp .env.example .env
docker compose up -d --build

Danach läuft die Oberfläche unter http://127.0.0.1:7000 — erreichbar von jedem Gerät im eigenen Netzwerk, auch vom Handy.

3.Ein Modell laden

In LM Studio ein Modell herunterladen (z. B. Qwen, Gemma, Llama oder GLM), es im Reiter „Developer" laden und den lokalen Server starten. LM Studio zeigt dann eine Adresse wie http://192.168.x.x:1234.

In Odysseus über das Zahnrad einen Modell-Server hinzufügen, diese Adresse mit angehängtem /v1 eintragen und auf „Test" klicken. Sobald „online" erscheint, ist das Modell verbunden.

4.Loslegen

Neuen Chat öffnen, das geladene Modell auswählen — und ihr habt drei Modi: normaler Chat, Agenten-Modus für mehrschrittige Aufgaben und Web-Suche über die mitgelieferte SearXNG-Instanz. Alles auf eurer Maschine.

Modellwahl

Welches Modell nehmen?

Es gibt keinen Königsweg — die richtige Wahl hängt an eurer Hardware. Ein paar bewährte Open-Weight-Familien als Startpunkt:

  • Gemma (Google) und Llama (Meta) — solide Allrounder, gut dokumentiert.
  • Qwen (Alibaba) — stark im Schlussfolgern, viele Größen von klein bis groß.
  • GLM und Mistral — gute Alternativen, wenn die anderen zu groß sind.

Odysseus bringt einen Cookbook-Bereich mit, der aus über 270 Modellen passende Empfehlungen für eure Hardware vorschlägt. Praktisch, wenn ihr nicht raten wollt, was auf eure GPU passt. Fangt klein an (7–8 Mrd. Parameter) und tastet euch hoch, bis Geschwindigkeit und Qualität für euch stimmen.

Von überall

Dein Modell von unterwegs nutzen — sicher

Bis hierhin läuft alles im eigenen Netzwerk. Die neue LM-Studio-Funktion LM Link erweitert das auf „von überall" — ohne dass ihr dafür irgendetwas ins offene Internet stellen müsst.

Wie es funktioniert: LM Link verbindet eure eigenen Geräte über eine Ende-zu-Ende-verschlüsselte Verbindung, aufgebaut in Partnerschaft mit Tailscale. Die Geräte finden sich automatisch, egal in welchem Netz sie gerade hängen. Weder Tailscale noch LM Studio sehen dabei eure Prompts oder Antworten.

Der Clou: Ein entferntes Modell verhält sich wie ein lokales — es bleibt unter localhost:1234 erreichbar. Heißt konkret: Das schwere Modell läuft auf eurer GPU-Maschine zuhause, ihr arbeitet vom Laptop unterwegs, und in Odysseus müsst ihr nichts umstellen.

Einrichten: Im LM-Studio-Desktop genügen ein paar Klicks über „Add a remote machine"; alternativ per Terminal mit lms login und anschließend lms link enable. LM Link läuft derzeit in einer kostenlosen Preview — kostenpflichtige Pläne sind für den späteren Release angekündigt.

Ehrlich bleiben

Grenzen & Sicherheit

Lokale KI ist beeindruckend, aber kein fertiges Produkt. Was ihr wissen solltet, bevor ihr euch darauf verlasst:

Modelle halluzinieren. Lokale Modelle erfinden Fakten genauso wie die großen — bei kleineren Modellen eher mehr. Im Video mischte das Modell schlicht falsche Figuren in eine Antwort. Behandelt keine Ausgabe als gesichert; prüft jede Behauptung und jede Quelle nach.

Die Codebasis ist jung. Odysseus ist in wenigen Tagen mit viel KI-Hilfe entstanden — über 800 Pull Requests, bekannte Architektur- und Sicherheitslücken. Es ist ein spannender Startpunkt zum Experimentieren, kein gehärtetes System.

Nicht ins offene Internet hängen. Lasst Odysseus auf localhost oder im Heim-/Büro-Netz laufen. KI-generierter Code bringt überdurchschnittlich viele Schwachstellen mit — kein Port-Forwarding nach draußen. Wollt ihr von unterwegs ran, nehmt LM Link (siehe oben): verschlüsselt und ohne offene Ports.

Krypto-Spam ignorieren. Rund um populäre Open-Source-Projekte tauchen Betrüger auf, die Tokens oder „Airdrops" pushen. Das hat mit dem Projekt nichts zu tun. Finger weg.

Und zur Einordnung: Odysseus ist nicht das erste Tool dieser Art. Wer etwas Ausgereifteres sucht, schaut sich Open WebUI zusammen mit Ollama an — gleiches Prinzip, längere Reifezeit. Der eigentliche Punkt von Odysseus ist nicht, dass es neu ist, sondern dass es zeigt, wie zugänglich lokale KI geworden ist.

Die goldene Regel

Lokal heißt nicht magisch. Ihr gewinnt Datenhoheit, keine Wahrheit. Das Setup ändert nichts daran, dass ihr die Ergebnisse prüfen müsst — es ändert nur, wer eure Daten sieht: niemand außer euch.

Die Kombination macht's: lokales Modell für die Privatsphäre, lokale Suche für aktuelle Quellen, euer eigenes Urteil für den Rest.

Für die Lehre: 3 Übungsideen

1. Privacy-by-Design live 30 Min

Setzt mit den Studierenden gemeinsam Odysseus auf einem Rechner auf. Der Aha-Moment: Eine vollwertige KI-Oberfläche, die nachweislich offline funktioniert — Netzwerkstecker ziehen, Chat läuft weiter.

2. Lokal vs. Cloud 20 Min

Dieselbe Frage an das lokale Modell und an ChatGPT stellen. Antworten, Geschwindigkeit und Quellen vergleichen. Diskussion: Wo lohnt sich die Cloud, wo reicht lokal — und was war der Preis dafür?

3. Halluzinationen jagen 25 Min

Studierende stellen dem lokalen Modell Fachfragen aus dem eigenen Gebiet und prüfen jede Antwort gegen echte Quellen. Ziel: ein Gespür dafür entwickeln, wo Modelle zuverlässig sind und wo sie raten.
